# Prior Knowledge: Cloud Security Engineer with IAM & Threat Modelling Depth

4 years of hands-on public cloud security engineering, with demonstrated depth in IAM, security architecture review, threat modelling, and risk assessment. This sets a high floor for Domain 5 (IAM) and parts of Domain 3 (Security Architecture) and Domain 1 (Risk Management). All other CISSP domains should be treated as new territory until evidence of understanding appears.

**Evidence**: Stated directly in session 1 context.

**Implications**:
- Skip introductory IAM concepts in Domain 5 lessons; test for depth, then fill gaps
- Domain 1 risk management vocabulary (ALE, SLE, AV, EF) may be known by practice but not by CISSP label — watch for this mismatch
- Network security (Domain 4), physical security (Domain 3 Ch 10), legal/compliance (Domain 1 Ch 4), and software dev security (Domain 8) are the highest-priority knowledge gaps to close
- Exam mindset shift will be the main challenge: moving from "how to configure" to "what should a manager decide first"
