# Mission: SC-500 — Implementing End-to-End Security Controls for Cloud and AI Workloads

## Why
Pass the SC-500 exam in 15 weeks to validate your ability to implement comprehensive security controls across Azure cloud and AI workloads — translating hands-on lab skills into a recognised engineering credential that strengthens your security review practice.

## Success looks like
- Pass the SC-500 exam with a score of 700 or above
- Configure and secure Microsoft Entra ID, Key Vault, and Azure RBAC from scratch
- Implement NSGs, Private Endpoints, Azure Firewall, and Virtual WAN security
- Harden VMs, containers, AKS, and App Services against real attack vectors
- Secure AI workloads using Defender for AI, Entra Agent ID, and Foundry guardrails
- Monitor security posture and respond to alerts using Defender for Cloud and Microsoft Sentinel
- Complete at least one hands-on lab per domain before sitting the exam

## Constraints
- **Time**: 15 weeks, 8 hours/week (120 hours total)
- **Resources**: Pluralsight (full access) + Microsoft Learn sandboxes (primary); $100/month Azure subscription as backstop for labs that require persistent resources
- **Prior knowledge**: Azure beginner; small foothold in identity (Entra ID basics)
- **Preference**: Free-first approach — maximise Pluralsight and MS Learn before spending Azure credits

## Out of scope
- AZ-900 / AZ-104 fundamentals (covered only where SC-500 requires it)
- Deep DevSecOps pipelines (SC-400/AZ-400 territory)
- On-premises Active Directory deep-dives beyond what Entra hybrid covers
- Cost optimisation / FinOps (not examined)

## Learning Order (15-week plan)

| Weeks | Track | Exam Domain |
|-------|-------|-------------|
| 1–3   | Identity, Access & Governance | Manage identity, access, and governance (20–25%) |
| 4–6   | Networking & Storage Security | Secure storage, databases, and networking (25–30%) |
| 7–9   | Compute Security | Secure compute — servers, containers, app platforms (20–25%) |
| 10–11 | AI Security | Secure compute — AI workloads sub-track |
| 12–13 | Security Posture & Sentinel | Manage and monitor security posture (20–25%) |
| 14–15 | Mock exams & interleaved review | All domains |
