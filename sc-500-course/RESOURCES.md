# Resources: SC-500 — Azure Cloud & AI Security

## Knowledge

### Official Microsoft Sources

- [SC-500 Exam Study Guide](https://learn.microsoft.com/en-us/credentials/certifications/resources/study-guides/sc-500)
  The authoritative skill outline. Use as the master checklist — every bullet is a potential exam question. Re-read at the start of each new track.

- [Microsoft Entra ID Documentation](https://learn.microsoft.com/en-us/azure/active-directory/)
  Primary reference for identity, PIM, conditional access, managed identities, and app registrations.

- [Microsoft Defender for Cloud Documentation](https://learn.microsoft.com/en-us/azure/defender-for-cloud/)
  Covers CSPM, workload protection plans, and security recommendations. Use for posture management track.

- [Microsoft Sentinel Documentation](https://learn.microsoft.com/en-us/azure/sentinel/)
  Data connectors, analytics rules, automation (SOAR), and workbooks. Use for the posture/Sentinel track.

- [Azure Key Vault Documentation](https://learn.microsoft.com/en-us/azure/key-vault/)
  Keys, secrets, certificates, access policies, firewall rules. Use during Identity track.

- [Azure Firewall Documentation](https://learn.microsoft.com/en-us/azure/firewall/)
  DNAT, network rules, application rules, policy management. Use during Networking track.

- [Azure Policy Documentation](https://learn.microsoft.com/en-us/azure/governance/policy/)
  Built-in and custom policy definitions, compliance evaluation, initiative assignments.

- [Microsoft Learn — SC-500 Learning Paths](https://learn.microsoft.com/en-us/credentials/certifications/exams/sc-500#two-ways-to-prepare)
  Free, interactive modules with sandbox labs. Use as the primary hands-on practice vehicle.

### Video Courses

- [Pluralsight — SC-500 Learning Path](https://www.pluralsight.com/paths/sc-500)
  ⭕ To explore — use as primary video instruction source. Watch before labs for each domain.

### Reference Architecture

- [Azure Security Benchmark](https://learn.microsoft.com/en-us/security/benchmark/azure/overview)
  Controls-based security guidance mapped to CIS, NIST. Useful context for governance track.

- [Microsoft Security Best Practices](https://learn.microsoft.com/en-us/security/compass/microsoft-security-compass-introduction)
  High-level architecture guidance from the Microsoft CISO team.
